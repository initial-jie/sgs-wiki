## 一、机制(已确认)

裴秀三技围绕「地图」系统:

- **茂著(锁定技)**:回合开始时,或你画完一幅地图的**所有城市**时 → 手牌花色补齐到四色都有 + **展开一幅新地图**。回合结束时 → 从**本回合已展开地图产生的技能池**里**随机 3 个,你三选一**,获得的技能**持续到你下回合结束**。
- **尽览**:回合内你使用一张 ♠/♥/♣/♦ 牌后 → 沿对应方向把该方向所有地图**"走到底"(推箱子式滑到边)**;**每经过一座城市,立即执行该城市的图标效果**。
- **采风**:出牌阶段每幅地图限一次,弃任意张牌 → 从牌堆/弃牌堆随机换回等量的**其余花色**牌。

**花色 → 方向**:♠东/右 · ♥西/左 · ♣南/下 · ♦北/上。

### 已确认规则
1. **展开地图 = 随机抽**;回合内不重复(16 张全跑完才循环),跨回合可重复。
2. **地图起始格固定**(每州一个固定起点,需从截图确认)。
3. **每座城市 = 两层含义(都有)**:
   - **图标效果(经过即执行 / 尽览)**:走到该城市立刻结算。三类图标(梁州已印证):`摸N牌`(牌图标)、`回复N体力`(绿勾玉)、`朝某方向移N格`(箭头,到达特定格 —— 会改变走位)。
   - **城市技能(命名技能)**:经过该城市即"解锁"这条命名技能,使其进入结束阶段的三选一池。
4. **结束阶段三选一的池子** = 本回合 **①展开过的每个"州"的州级地图技能** + **②本回合实际经过(解锁)的每座城市的技能**。从这个池随机 3 个 → 你选 1 → 持续到下回合结束。
   - 👉 **这解答了"州奖励什么时候给"**:州级技能**没有单独的发放时刻**——它和"已解锁城市技能"一起进结束阶段的三选一池。展开某州 = 该州技能入池;经过某城 = 该城技能入池;回合结束一起随机 3 选 1。所以录像里看不到"完成地图瞬间给州奖励"是对的,本来就没有那个瞬间。
   - ⚠️ 因此严格说池子不是"该图全部 4 城",而是**你实际走到的城** + 州技(没走到的城不入池)。

## 二、坐标系 + 数据 Schema

**坐标系(用户 2026-07-14 定)**:`[x, y]`,**`[0,0]=左下角**;第一位 x=横坐标(0→右),第二位 y=纵坐标(0→上)。

每州结构(最终进 `prototype/shared/peixiu-maps.json`):

```jsonc
{
  "梁州": {
    "pinyin": "Liang",
    "grid": { "w": 5, "h": 5 },              // 网格宽(x)×高(y)
    "start": [3, 2],                          // 裴秀固定起点 [x,y] —— ⚠ 待用户确认梁州起点
    "walls": [[0,0],[0,1],[0,2],[0,3],[0,4],[1,0],[1,1],[1,4],[2,0],[4,0],[4,2],[4,3],[4,4]],
    "cities": [                               // 四城:坐标 + 经过图标 + 解锁的命名技能
      { "no":1, "name":"绵竹", "pos":[1,2], "icon":"heal:1",     "skill":"结束阶段，你观看牌堆顶三张牌，然后可以获得其中类型不同的牌各一张。" },
      { "no":2, "name":"巴西", "pos":[2,3], "icon":"draw:1",     "skill":"你可以指定一名其他角色，其当前手牌无次数限制。" },
      { "no":3, "name":"汉中", "pos":[3,4], "icon":"move:down:3","skill":"你可以与一名其他角色交换手牌。" },
      { "no":4, "name":"涪陵", "pos":[4,1], "icon":"draw:2",     "skill":"你的装备牌不能被其他角色弃置。" }
    ],
    "stateSkill": "你成为【杀】的目标后，使用者弃置一张牌。"   // 州级技能(与城市技能一起进结束阶段池)
  }
}
```

**图标编码(icon)约定** —— 已由梁州印证:

| icon 编码 | 图标 | 含义 |
|---|---|---|
| `draw:N` | 牌图标 + 角标 N | 经过时摸 N 张牌 |
| `heal:N` | 绿色勾玉 + 角标 N | 经过时回复 N 点体力 |
| `move:<方向>:N` | 箭头 + 角标 N | 到达时立刻朝该方向自动移动 N 格(方向 up/down/left/right) |
| … | 其余图标 | 后续州若出现新图标,再补 |

## 四、工具形态(数据齐后开工)

- **带网格棋盘**(用户选定)。展开地图 → 棋盘按 Schema 画当前州(格子/墙/四城/固定起点/图标)+ 侧栏列已解锁城市技能。
- 玩家点方向(♠东/♥西/♣南/♦北),工具**模拟推箱子走位**(滑到墙 + 经过城市自动执行图标含 move 类)+ 追踪四城画完(**不含最优解 solver**,最优走位靠玩家/peixiu)。
- 四城全画完 → 亮「下一张(随机,回合内不重复)」。
- **结束阶段** → 从"本回合州技 + 已解锁城市技能"池随机 3 抽 1 → 记录留到下回合的技能。
- 接入范式同其他工具(room-logic + room.html view/bind + tools/peixiu.html 单人版 + index.html 卡 + generals.json tool 映射);数据源 `peixiu-maps.json`。
